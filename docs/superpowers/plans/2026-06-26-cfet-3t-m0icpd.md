### Task 5: Power-row routing ban

**Files:**
- Modify: `src/cellgen/archit/CFET/main.py`

- [ ] **Step 1: Add helper** (near `_ban_other_nets_on_pwr_columns`)

```python
def _ban_signal_on_power_rows(self):
    if self.c_tech.power_config != "M0ICPD":
        return
    if not self.power_row_indices:
        raise ValueError("M0ICPD requires power_row_indices")
    forbidden_edges = OrderedDict()
    for layer_name in ("M0", "PC", "BPC"):
        layer_idx = self.lgg.layer_to_idx[layer_name]
        for ri in self.power_row_indices:
            row = self.lgg.row_in_layer(layer_name, ri)
            for edge in self.edge_vars:
                u, v = edge
                if (u[0] == layer_idx and u[1] == row) or (v[0] == layer_idx and v[1] == row):
                    forbidden_edges[edge] = self.edge_vars[edge]
    for edge_var in forbidden_edges.values():
        self.opt.Add(edge_var == 0)
```

This bans top-view power rows 0 and 5 on M0/PC/BPC. Power rails are drawn by
GDS; power/ground nets are not routed through normal signal flow.

### Task 6: GDS in-cell power rails

**Files:**
- Modify: `src/cellgen/postprocess/gds_CFET_SH.py`

- [ ] **Step 1: Add `__m0_icpd__` method**

Draw full-width M0 boxes at row 0 and row 5 y-coordinates. One fine row is one
0.5T band, so the band size is `m0_pitch`:

```python
def __m0_icpd__(self):
    band = self.tech_data.m0_pitch
    rail_thickness = min(self.tech_data.m0_power_rail_thickness, band)
    width = self.tech_data.cp_pitch * self.tech_data.col
    # VSS — bottom 0.5T (fine row 0)
    ly_vss = (band - rail_thickness) / 2
    box = Box(0, ly_vss, width, ly_vss + rail_thickness).pyabox_obj()
    self.cell.shapes(self.M0_layer_idx).insert(box)
    # VDD — top 0.5T (fine row 5)
    ly_vdd = self.height - band + (band - rail_thickness) / 2
    box = Box(0, ly_vdd, width, ly_vdd + rail_thickness).pyabox_obj()
    self.cell.shapes(self.M0_layer_idx).insert(box)
```

Use `band = self.tech_data.m0_pitch`, because one fine row is one 0.5T band.
M0ICPD GDS height must be `self.tech_data.m0_pitch * self.tech_data.track * 2`.
M0ICPD route row transforms use raw fine-row coordinates; M0BPR keeps legacy
`row / SOLVER_RESCALE`.
